Exercices – Automates cellulaires en dimensions 1+1.

Dans cet exercice, vous allez implémenter un automate cellulaire unidimensionnel en Python.

Nous allons travailler avec deux modules :

  • automaton_sim : logique de simulation (cellules, ligne, mise à jour)
  • automaton_display : affichage texte dans le terminal

Objectif

On considère une ligne de cellules, chacune pouvant être :

  • vivante
  • morte
  • une cellule spéciale de bordure

L’état d’une cellule à l’instant suivant dépend :

  • de l’ensemble des cellules voisines définies par une liste offset d’offsets,
  • d’une règle : si le nombre de voisins vivants appartient à un ensemble donné, la cellule devient vivante.

Exemple de voisinages

Quelques exemples de voisinages possibles :

  • Voisinage classique gauche/droite : offset = [-1,1].
  • Voisinage qui contient la cellule elle-même : offset = [-1,0,1].
  • Voisinage étendu : offset = [-2,-1,0,1,2].

Exercice 0 — Classe Cell

Implémentez une classe Cell.

Spécification

  1. Le constructeur __init__(state) prend une chaîne parmi : "alive", "dead", "border".

  2. La méthode is_alive() renvoie True si la cellule est vivante, False sinon.

  3. La méthode set_state(state) permet de modifier l’état de la cellule.

Les cellules de bordure sont toujours considérées comme mortes.

Exercice 1 — Classe Line

Implémentez une classe Line qui représente une ligne de cellules.

Spécification

  1. __init__(self, n) crée une ligne de longueur n avec uniquement des cellules mortes.

  2. set_alive(self, i) rend vivante la cellule d’indice i si elle existe.

  3. get(self, i) renvoie :

    1. la cellule d’indice i si i est valide,
    2. une cellule "border" sinon.

  4. copy(self) renvoie une copie de la ligne.

Exercice 2 — Voisinages et règles

Nous voulons simuler une ligne de cellules selon une règle basée sur un voisinage choisi.

Implémentez la fonction suivante :

def update_all(line, offsets, alive_counts):
  """
  line : objet Line
  offsets : liste d'entiers (ex: [-1, 1])
  alive_counts : ensemble d'entiers (ex: {1, 2})

  Si le nombre de voisins vivants est dans alive_counts,
  la cellule devient vivante, sinon elle est morte.
  """

Exercice 3 — Affichage

Dans un second module automaton_display.py, écrivez une fonction : render(line) qui affiche une ligne line de cellules en utilisant :

  • █ (ou un autre symbole) pour une cellule vivante,
  • espace (ou un autre symbole) pour une cellule morte.

Exercice 4 — Simulation

Écrire un programme principal main.py qui :

  • Crée une ligne de taille N=100.

  • Active uniquement la cellule centrale.

  • Affiche 40 générations.

Utilisez un voisinage et une règle de votre choix.
